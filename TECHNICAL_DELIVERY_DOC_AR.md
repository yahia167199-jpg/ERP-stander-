# وثيقة النشر والتسليم التقني (Technical Deployment & Delivery Document)
## مشروع: نظام ERP المتكامل لإدارة الموارد

### 1. نظرة عامة على النظام (System Overview)
نظام ERP متكامل مبني لخدمة الشركات والمؤسسات الصغيرة والمتوسطة، يهدف إلى أتمتة العمليات الأساسية مثل المشتريات، المبيعات، إدارة المخازن، والعمليات الحسابية. يتميز النظام بواجهة مستخدم ثنائية اللغة (عربي/إنجليزي) ويدعم الوضع الداكن (Dark Mode).

### 2. البنية التقنية (Tech Stack)
*   **الواجهة الأمامية (Frontend):** React 19 (مع Vite كأداة بناء).
*   **التصميم والأنماط (Styling):** Tailwind CSS v4 للمرونة والسرعة في التصميم.
*   **التحريكات (Animations):** Motion (framer-motion سابقاً).
*   **قاعدة البيانات (Database):** Google Firebase Firestore (قاعدة بيانات NoSQL في الوقت الفعلي).
*   **الهوية والوصول (Authentication):** Firebase Authentication (دعم تسجيل الدخول بواسطة Google).
*   **تخزين الملفات (Storage):** Firebase Storage (لتخزين الصور والمرفقات).
*   **الرسوم البيانية:** Recharts لتمثيل البيانات إحصائياً.
*   **أدوات إضافية:** Excel (XLSX) لتصدير التقارير، Date-fns لمعالجة التواريخ.

### 3. الوحدات البرمجية المنجزة (Implemented Modules)
1.  **لوحة التحكم (Dashboard):** عرض إحصائي شامل للمبيعات، المشتريات، وتوافر المنتجات.
2.  **قسم الكاشير (Sales Point):** واجهة سريعة للبيع مع دعم ماسح الباركود.
3.  **المخزون والمنتجات:**
    *   إدارة المنتجات (يدعم أنواع القطع: قطعة واحدة أو زوج).
    *   إدارة المستودعات المتعددة.
    *   تحويلات المخزون بين المستودعات.
    *   عمليات الجرد (Stocktaking).
4.  **قسم التصنيفات:** نظام شجري للتصنيفات (رئيسي وفرعي).
5.  **المشتريات والموردين:** تتبع فواتير الشراء وإدارة بيانات الموردين.
6.  **المبيعات والعملاء:** تتبع فواتير البيع وإدارة بيانات العملاء.
7.  **المرتجعات:** معالجة مرتجع المبيعات والمشتريات مع ربطها بالفواتير الأصلية.
8.  **قسم الفواتير (جديد):** سجل موحد لجميع الفواتير يربط العمليات (بيع/شراء) بمرتجعاتها مع إمكانية الطباعة.
9.  **المحاسبة:** سجل المعاملات المالية (دخل/مصروف) وتوليد التقارير.
10. **إعدادات النظام:** التحكم في بيانات الشركة، تصميم الفاتورة، وإدارة المستخدمين والصلاحيات.

### 4. نموذج البيانات (Data Model)
يعتمد النظام على المجموعات (Collections) التالية في Firestore:
*   `products`: بيانات المنتجات والمخزون.
*   `suppliers`: بيانات الموردين.
*   `customers`: بيانات العملاء.
*   `sales`: فواتير المبيعات.
*   `purchases`: فواتير المشتريات.
*   `returns`: عمليات المرتجع.
*   `transactions`: العمليات المالية (Accounting).
*   `warehouses`: تعريف المستودعات.
*   `categories`: شجرة التصنيفات.
*   `users`: ملفات المستخدمين وصلاحياتهم.
*   `settings`: إعدادات النظام (اسم الشركة، الضرائب، إلخ).

### 5. نظام الأمان والصلاحيات (Security & RBAC)
*   تم تطبيق **قواعد أمان Firestore (Security Rules)** التي تضمن عدم وصول أي مستخدم لبيانات لا يملك صلاحية عليها.
*   **صلاحيات حسب الواجهة:** يمكن للمدير تحديد الشاشات التي يمكن لكل مستخدم رؤيتها.
*   **صلاحيات CRUD:** التحكم في قدرة المستخدم على (الإضافة، التعديل، الحذف) لكل وحدة على حدة.
*   **بوابة الوصول (Gate Security):** إمكانية وضع اسم مستخدم وكلمة مرور إضافية لكل شاشة لتعزيز الأمان.

### 6. تعليمات التشغيل والنشر (Deployment Instructions)
1.  **متطلبات التشغيل:** Node.js v18+.
2.  **تثبيت التبعيات:** `npm install`.
3.  **التشغيل في بيئة التطوير:** `npm run dev`.
4.  **البناء للإنتاج:** `npm run build` (سيتم توليد ملفات ساكنة في مجلد `dist`).
5.  **تهيئة Firebase:**
    *   يجب وضع ملف الإعدادات `firebase-applet-config.json` في المجلد الرئيسي.
    *   نشر القواعد عبر `npm run deploy-rules` (إذا تمت تهيئتها).

### 7. التسليم الفني (Technical Handover)
*   الكود المصدري منظم ومكتوب بلغة **TypeScript** لضمان استقرار الأنواع وتقليل الأخطاء.
*   النظام جاهز للاستخدام الفوري بعد ربطه بحساب Firebase الخاص بالعميل.
*   جميع الواجهات مستجيبة (Responsive) وتعمل على الهواتف والأجهزة اللوحية والحواسيب.
